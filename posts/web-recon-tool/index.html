Web Recon Tool

While researching various bounty programs on bugcrowd, I started looking into tools and methodology other bug hunters were using when performing recon on a program. That research led to this tool.

This tool is inspired by OSCP tools like AutoRecon or nmapAutomator. I figured creating one of my own would present a good opportunity to practice scripting in Python and containerization while learning about popular bug hunting tools.

Using the tool

In this example I’m scanning Tesla.com

git clone https://github.com/fr4z/web-recon.git
cd web-recon/
./run.sh tesla.com
  1. he script will build the directory structure which will store results:
tesla.com
|-- assetfinder.txt
|-- httprobe.txt
|-- gowitness
    |--tesla.com.png
  1. Then the script will look for subdomains using assetfinder.
  2. Check which of those subdomains are alive using httprobe
  3. Take a screenshot of each of the sites from list of live subdomains with gowitness

Future Plans

So far the tool takes one domain, enumerates subdomains and takes a screenshot of each one rendered. Some plans for the future:

  1. Run these tools concurrently
  2. Add more tools like gobuster, massscan and subjacker.
  3. Make the output fancier
  4. Incorporate some OOP
